## 2026-07-08 v0.13.0b0 re-audit note

Deep re-audit at HEAD `6753af3` with the v0.13.0b0 Solar Surplus feature uncommitted in the working tree. Plans 010-016 target that new code. **Executor note**: because the code being planned against is uncommitted, the drift check in each plan will report large diffs against the "Planned at" SHA; executors should compare the plan's "Current state" excerpts against the live working-tree file instead of relying on the diff line count. The excerpts were captured from the live tree at plan-write time.

Rejects unique to this audit run (in addition to the ones already listed below):

- **`except A, B:` "Python 2 syntax" claim in 6 files**. Python 3.14 PEP 758 (accepted for 3.14) allows unparenthesized `except A, B:` and treats it as the tuple form `except (A, B):`. Verified in the venv (`.venv/bin/python -c "..."`) — both types are caught identically. False positive from subagents trained on pre-3.14 Python.
- **Race on `self._solar_surplus_unavailable` set mutation** in the `asyncio.gather` fan-out. asyncio is cooperative; there is no `await` between the set-check and set-mutation inside `_note_solar_unavailable`. Not a real race.
- **DST-unsafe `_solar_next_hour_boundary`**. ENGIE returns timezone-aware ISO strings with explicit offset; `datetime.fromisoformat` + `.astimezone(UTC)` is DST-correct. Not a bug.
- **Soft-fail solar flag to `True` on transient errors is unsafe**. Deliberate design decision documented in-session. The alternative (soft-fail to `False`) strips entities on transient outage. Not a bug.
- **`runtime.reload_pending` race between concurrent subentries**. HA's config-entry state machine deduplicates concurrent reload requests; even if the debounce is racy, the downstream reload is idempotent. Plans 015 + 016 lock in the intended per-refresh debounce; that's sufficient.
- **`ZoneInfo(EPEX_TZ)` instantiated per property read**. `zoneinfo.ZoneInfo` uses a stdlib cache; repeated calls with the same key return the same object. Effectively free.
- **Path traversal via `delivery_point_id` in solar-surplus URL**. Input is `f"{ean}_ID1"` from `service_points` populated by ENGIE's own API; not attacker-controllable.
- **Unmasked BAN in solar-flag warning log**. Coordinator uses `mask_identifier(business_agreement_number)` at `coordinator.py:467`; the finding was hallucinated.
- **`async_update_reload_and_abort` deprecation**. Code already uses `async_update_and_abort()` per CLAUDE.md directive; no unfixed regression.
- **Real-time measured solar injection API**. LOW-confidence speculation; no evidence ENGIE exposes this endpoint.
- **Untracked dev log files at repo root** (`debug(1).log.gz`, `logo-pack(1)/`). Real minor clutter, but skipped by the user in the plan selection.
- **Solar surplus automation blueprint (DOC-01)** and **README solar automation example (DOC-02)**. Skipped by the user in the plan selection; can be revisited on request.
- **Expose Happy Hours previous-month comparison metrics (DIR-01)**. Grounded direction finding (three % metrics fetched but discarded); skipped by the user in the plan selection; can be revisited on request.
