## Why this matters

`__init__.py` has 8+ `# type: ignore[attr-defined]` markers hanging off `call.data.get(...)` accesses in the two service handlers (`_handle_import_history` and `_handle_clear_import_history`). The reason is that we type `call` loosely as `object` to avoid pulling `ServiceCall` into the file's import surface. The cost: silent type breakage if the service schema changes, no IDE assistance on what keys exist on `call.data`, and 8 redundant markers cluttering the code. The right fix is to type `call: ServiceCall` and extract the fields into a small dataclass at the top of each handler — one validated boundary, no ignore markers downstream.

### Home Assistant service schema

Services defined in `services.yaml` do NOT enforce type constraints at the Python side by default (HA's service schema is dict-shaped, not TypedDict-shaped). So the extractor is defensive, not driven by an autogenerated schema. The type of `call.data` from HA core is `ReadOnlyDict` (a `Mapping[str, Any]`), so any `# type: ignore` was a workaround for `object` typing, not a real schema mismatch.

### Repo conventions

- Match the terse present-tense docstring style used throughout.
- Prefer `@dataclass(frozen=True, slots=True)` for small immutable extractors (see `_StreamSpec` in `_statistics.py` for the pattern).
- Keep the extractor local to `__init__.py` — no new module.

### Step 2: Define a small dataclass per service

Immediately above `_async_register_services` (roughly wherever local helpers currently live), add:

```python
@dataclass(frozen=True, slots=True)
class _ImportHistoryCall:
    """Extracted, validated payload of an `engie_be.import_history` service call."""

    device_ids: list[str]
    energy_type: list[str] | None
    include_costs: bool
    start_date: date | None
    end_date: date | None

    @classmethod
    def from_service_call(cls, call: ServiceCall) -> _ImportHistoryCall:
        """Read the fields off ``call.data`` with the same defaults the handler used inline."""
        data = call.data
        return cls(
            device_ids=list(data.get(ATTR_DEVICE_ID) or []),
            energy_type=data.get(ATTR_ENERGY_TYPE),
            include_costs=bool(data.get(ATTR_INCLUDE_COSTS, False)),
            start_date=data.get(ATTR_START_DATE),
            end_date=data.get(ATTR_END_DATE),
        )


@dataclass(frozen=True, slots=True)
class _ClearImportHistoryCall:
    """Extracted, validated payload of an `engie_be.clear_import_history` service call."""

    device_ids: list[str]
    energy_type: list[str] | None
    include_costs: bool

    @classmethod
    def from_service_call(cls, call: ServiceCall) -> _ClearImportHistoryCall:
        data = call.data
        return cls(
            device_ids=list(data.get(ATTR_DEVICE_ID) or []),
            energy_type=data.get(ATTR_ENERGY_TYPE),
            include_costs=bool(data.get(ATTR_INCLUDE_COSTS, False)),
        )
```

Notes:
- `dataclass` and `date` must be importable in this file. `date` is likely already imported (grep to confirm). `dataclass` is stdlib — add `from dataclasses import dataclass` to the imports if missing.
- Use `list()` around `data.get(ATTR_DEVICE_ID) or []` to freeze the input (HA passes an immutable Mapping).
- `slots=True` matches the existing `_StreamSpec` pattern in `_statistics.py`.

### Step 3: Rewrite the two handlers to use the extractors

**`_handle_import_history`** — change signature and body:

```python
async def _handle_import_history(call: ServiceCall) -> None:
    payload = _ImportHistoryCall.from_service_call(call)
    if not payload.device_ids:
        raise ServiceValidationError(
            translation_domain=DOMAIN,
            translation_key="service_no_target_device",
        )
    if payload.energy_type is not None and not payload.energy_type:
        raise ServiceValidationError(
            translation_domain=DOMAIN,
            translation_key="service_no_energy_type_selected",
        )
    streams = streams_for_energy_types(
        payload.energy_type,
        include_costs=payload.include_costs,
    )
    LOGGER.debug(
        "import_history called: device_ids=%s energy_type=%s"
        " include_costs=%s start=%s end=%s",
        payload.device_ids,
        payload.energy_type,
        payload.include_costs,
        payload.start_date,
        payload.end_date,
    )
    # ... rest of the handler, using payload.start_date / payload.end_date / payload.device_ids
```

Replace every `call.data.get(...)` with the corresponding `payload.<field>`. Drop every `# type: ignore[attr-defined]` marker on those lines.
